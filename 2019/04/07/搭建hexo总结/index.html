    搭建Hexo总结

    先说说我的参考文档吧,搭建Hexo一开始看对我比较有用的文档是
    hexo史上最全搭建教程
    该篇文章中有几个点是需要注意的,下文稍后会提起.

    另外一个是Hexo + GitHub (Coding) Pages 搭建博客
    这篇文章讲了很多拓展的用法.

    下面我就讲一下我遇到的一些坑,和一些推荐.

    1. 不推荐直接在私有云服务上进行部署,因为中间涉及到体检和更新,不是很推荐.–毕竟懒,这种繁琐的更新,个人不太能接受
    2. 使用github进行部署时,创建名称一定要是xxxx(githubName).github.io(com好像也可以),不然下面生成的github中page生成后会形成二级域名,导致js和css等无法错误.
    3. 配置自己的域名的时候
      一定要在github中custom domain的配置将头加进去
      如果你配置的blog 你就要像我一下

    第2点非常重要,一点要注意.
